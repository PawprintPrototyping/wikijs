## SNMP

[Slides](https://docs.google.com/presentation/d/1VXr8ikwck-1PlbhxyObK8lnMoRMS17IZ-UqKcpjpH7w/edit?usp=sharing)

- Widely used for network management and monitoring (observability).
- Exposes management data as variables organized in MIBs describing a device’s status and configuration.
- SNMP collectors/managers can query and manipulate these values.

- If you don’t need it, turn it off!
- Pay close attention to read-only configurations
- SNMP v1 uses plain text “auth” via the common community strings
  - “public” for read-only
  - “private” for read-write
- Use SNMP v3 authentication with strong secrets if available
- Use ACLs if available, though be aware that IP address spoofing attacks still possible.
- Consider DTLS or a separate management network

Network management suites poll devices for stats and offer some management capabilities, discoverability, etc.
Some examples:
- librenms - https://www.librenms.org/
- Zabbix - https://www.zabbix.com/
- OpenL2M - https://openl2m.readthedocs.io/
- Nagios - https://www.nagios.org/
- ntopng - https://www.ntop.org/
- Cacti - https://www.cacti.net/

More options: https://github.com/awesome-foss/awesome-sysadmin#monitoring

 - SNMP v1 is plaintext!  Act accordingly!!
 - Geo has some more notes on security:
     - Great as a red teamer to identify devices, services, peak load times, lots of goodies
     - Always opt for SNMP v3 with auth when possible
     - NetSNMP beware - lots of inadvertent pivot points especially on embedded devices, good for jailbreaking
 - `snmpwalk` tool good for walking the entire tree!

- [HomeAssistant has an SNMP integration](https://www.home-assistant.io/integrations/snmp)
